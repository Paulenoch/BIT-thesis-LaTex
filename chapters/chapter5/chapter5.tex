\chapter{全SSM架构探索：MambaVision替代的系统消融实验}

\section{实验目标与评价问题定义}

第3章确立了ViT+Mamba策略网络在高速端到端避障中的有效性，
第4章解决了流式部署一致性问题。
在此基础上，第5章聚焦于一个自然的架构演进方向：
能否将视觉编码器从ViT替换为基于SSM的MambaVision\cite{Hatamizadeh2025MambaVisionCVPR}，
从而构建空间--时间统一的全SSM架构？

替代动机包含三个层面。
第一，架构统一性：若空间编码与时序聚合均采用SSM范式，
整体架构将具有更一致的归纳偏置。
第二，分辨率适配：MambaVision的分层卷积嵌入对输入分辨率的适配更灵活，
为提升视觉输入精细度提供空间。
第三，OOD泛化潜力：MambaVision在ImageNet上的预训练权重可能赋予其更强的视觉泛化基础，
有望改善策略在分布外环境中的表现。

基于上述动机，第5章回答以下三个核心问题：
\begin{enumerate}
  \item 安全性：在保持控制闭环主体（时序Mamba模块、训练流程、部署协议）不变的条件下，
    MambaVision替代ViT是否导致闭环安全退化？
  \item 决定性因素：哪些设计与训练因素（归一化口径、输入分辨率、骨干解冻策略、结构改动、数据补充）
    对最终性能具有决定性影响？
  \item 部署性：最终模型的推理延迟是否仍满足实时闭环控制要求？
\end{enumerate}

为系统回答上述问题，
第5章采用逐步消融验证的实验方法学：
每一阶段仅引入一个设计变量的变化，
设定明确的通过/失败判据，
失败时记录归因并分流到替代路线。
这一方法学确保每一步变化可解释、可复现，
避免多变量同时变化导致的归因混淆。

\section{实验环境、数据与评测协议}

\subsection{评测环境与速度档}

第5章实验沿用第2章定义的统一评测协议（表~\ref{tab:eval_protocol_unified}）：
在Spheres（同分布）与Trees（分布外）两类环境中，
以$\SI{3}{m/s}$、$\SI{5}{m/s}$、$\SI{7}{m/s}$、$\SI{9}{m/s}$、$\SI{12}{m/s}$五档目标速度进行评测，
每档10次独立试验。
碰撞不终止回合，回合终止条件与第2章一致。
所有实验在NVIDIA RTX 4060 Laptop GPU上执行，
评测环境参数（障碍密度、随机种子等）在所有实验中保持固定，
以确保对比的公平性。

\subsection{指标体系}

除第2章定义的全程碰撞率（Collision Rate）、碰撞事件次数（Collision Count）、
指令抖动（Command Jerk）与推理延迟外，
第5章引入以下两项补充指标：

\begin{itemize}
  \item 单位距离碰撞密度（Collisions/meter, CPM）：
    碰撞事件次数除以回合内飞行的总距离。
    该指标用于排除"降低飞行速度换取安全"的伪改进——
    若模型通过减速来减少碰撞，
    则碰撞率下降但CPM不变或升高。
  \item 平均前向速度（Mean $V_x$）：
    $\bar{V}_x = \frac{1}{T}\sum_{t=1}^{T} v^x_t$，
    反映模型在保持安全的同时完成穿越任务的速度效率。
    该指标与CPM互补，共同构成"安全性--效率"的二维评价视角。
\end{itemize}

值得特别指出的是，
指令抖动（Jerk）在第5章中不作为安全性的代理指标。
后续实验将证明，Jerk降低并不必然意味着安全性提升，
"更平滑但更撞"的失败模式是真实存在的。

\section{深度归一化一致性验证}
\label{sec:ch5_norm}

在进行任何架构层面的消融实验之前，
本节首先验证一个影响全部后续结论可信度的前置条件：
深度图像归一化口径在训练阶段与评测阶段之间的严格一致性。
该验证属于实验控制变量的合规性检查（sanity check），
是确保后续所有消融对比在公平的输入分布下进行的必要前提。

\subsection{训练与评测归一化口径差异}

基线ViT+Mamba策略及所有MambaVision实验组在训练阶段均采用threshold归一化模式：
以固定阈值$\tau = 0.09$对原始深度值$d$进行归一化，
即$\hat{d} = \min(d / \tau,\; 1)$，
将有效深度范围线性映射至$[0, 1]$区间。
该配置赋予网络输入以明确的动态范围特性：
阈值$\tau$以内的近场障碍深度被线性展开至完整$[0, 1]$区间，
超出$\tau$的远场区域被截断至饱和值$1$，
使网络对近场障碍具有最高的距离区分灵敏度。

在将MambaVision接入现有训练管线的初期调试中，
发现评测脚本的深度归一化配置被错误地设置为png255模式，
即$\hat{d} = d_{\text{pixel}} / 255$，
将深度图的8-bit像素值直接除以$255$。
两种模式对同一深度场景产生的归一化值存在数量级差异：
threshold模式将$[0, \tau]$范围内的深度线性展开至完整$[0, 1]$区间，
而png255模式将同一范围压缩至$[0, \tau/255]$的极窄区间，
导致评测阶段的网络输入分布与训练阶段产生系统性偏离。

\subsection{输入分布偏移机制分析}

上述归一化口径差异在评测阶段引入了输入分布偏移（distribution shift），
其影响通过以下因果链传导至策略的闭环行为：

\begin{enumerate}
  \item 输入动态范围压缩。
    png255模式将训练阶段占据$[0, 1]$完整区间的深度输入
    压缩至接近零的极窄区间，
    输入特征的有效动态范围大幅收缩，
    网络各层接收到的激活值偏离其在训练阶段学习到的工作区间。
  \item 有效感知距离缩短。
    网络在训练阶段建立的深度值--物理距离映射关系被破坏。
    被压缩至近零区间的深度输入使网络丧失对近场障碍的精细距离判别能力，
    有效感知窗口严重退化。
  \item 控制指令幅度系统性衰减。
    由于输入特征偏离训练分布，
    策略网络的输出层产生幅度被系统性压低的速度指令，
    尤其是前向速度$v_x$分量受影响最为显著。
    该现象本质上是输入分布偏移导致的控制衰减
    （control attenuation due to input distribution shift），
    而非策略学习到的主动减速行为。
  \item 安全指标的虚假改善。
    前向速度降低导致单位时间内飞行距离缩短，
    碰撞暴露机会相应减少，
    碰撞率在中低速段表面上有所下降。
    然而，该"改善"并非源于控制鲁棒性的提升，
    而是速度退化的副产物。
    在高速段，速度退化过于剧烈时，
    任务完成能力直接受损，碰撞率不降反升。
\end{enumerate}

上述机制可概括为：
归一化口径偏移
$\rightarrow$ 输入动态范围压缩
$\rightarrow$ 控制指令幅度衰减
$\rightarrow$ 前向速度系统性降低
$\rightarrow$ 安全指标虚假改善。
若不识别并消除该混淆因素，
后续架构消融中碰撞率的任何变化都将无法正确归因于视觉编码器的设计差异。

高速段指令幅度更大，
分布偏移引起的比例性衰减效应更为显著。

在$\SI{5}{m/s}$段，

在$\SI{12}{m/s}$高速段，
分布偏移的后果更为严重且更具揭示性：
碰撞事件次数从$7.6 \pm 2.9$升至$15.0 \pm 4.7$（增幅$97.4\%$），
碰撞率从$0.042$升至$0.055$（增幅$31.0\%$）。
速度退化与碰撞率同时恶化，
说明分布偏移不仅压低了飞行效率，更实质性地损害了避障能力。

综合以上证据，
png255模式下碰撞率在中低速段的表面下降
本质上归因于速度退化导致的碰撞暴露减少，
而非控制鲁棒性的提升；
高速段碰撞率与碰撞次数的同时恶化
进一步证实，
归一化口径偏移对策略造成了全局性的性能损害。

\subsection{实验规范锁定}

上述验证确认：
归一化口径的训练--评测一致性是所有后续消融结论可信的必要前提。
归一化参数的任何偏差均会引入输入分布偏移这一混淆变量（confounding variable），
使碰撞率与速度效率的变化无法正确归因于视觉编码器的架构设计差异。

基于该结论，第5章确立以下实验规范并在此后所有阶段严格执行：
\begin{itemize}
  \item 归一化模式锁定为threshold，阈值$\tau = 0.09$，与训练配置完全一致；
  \item 评测日志中记录归一化元数据（模式、阈值、版本号）以供审计；
  \item 任何新实验在启动前须通过归一化配置一致性校验。
\end{itemize}

该合规性验证本身不构成架构层面的贡献，
但作为消融实验的前置控制变量检查，
它消除了输入分布偏移这一混淆因素，
确保后续各阶段的性能变化可严格归因于视觉编码器的架构差异。

\section{输入分辨率瓶颈验证}
\label{sec:ch5_resolution}

本节验证输入分辨率是否构成MambaVision在当前低分辨率配置下的性能瓶颈。
该实验在骨干完全冻结的条件下进行，
仅改变输入分辨率这一单一变量，
确保观测到的性能变化可严格归因于空间分辨率差异，
而非训练效果或骨干参数的影响。

\subsection{分辨率瓶颈分析}

基线ViT+Mamba使用$60\times90$的低分辨率深度图像作为输入。
ViT采用扁平化patch嵌入，将输入一次性切分为约21个token，
各token在此后的所有Transformer层中保持相同的空间粒度，
21个token已足以携带全局场景信息。

MambaVision-T\cite{Hatamizadeh2025MambaVisionCVPR}的架构与ViT存在根本差异：
其采用分层卷积嵌入（hierarchical convolutional embedding），
通过Stem层（累计$4\times$空间下采样）与三次Stage间$2\times$下采样，
在四个Stage上以递增的通道维度、递减的空间分辨率提取多尺度特征。
该设计使各Stage的空间token数量随深度指数级递减，
对输入分辨率的敏感性远高于ViT。

表~\ref{tab:ch5_token_analysis}给出了两种输入分辨率下各Stage的空间token数量。
在$60\times90$输入下，
Stage~3的特征图缩减至仅$3\times5=15$个token，
Stage~4进一步退化至$1\times2=2$个token——
此时空间信息几乎完全丧失，
全局平均池化（GAP）仅在2个空间位置上取均值，
骨干的空间表征能力被严重限制。
将输入提升至$128\times192$后，
Stage~4的token数量从$2$增至$4\times6=24$（$12\times$提升），
Stage~3从$15$增至$8\times12=96$（$6.4\times$提升），
为分层特征提取提供了充分的空间采样密度。

低分辨率下的空间采样不足对避障任务有直接影响。
深度图像中障碍物边界的空间频率较高，
当特征图分辨率过低时，
障碍物边界在分层下采样中被严重混叠（spatial aliasing），
网络无法精确判断障碍物的轮廓与距离，
导致避障决策精度下降。
因此，对于采用分层卷积嵌入的MambaVision而言，
输入分辨率不仅影响信息量，
更决定了分层特征提取机制能否正常工作——
低分辨率下的空间采样不足以支撑分层嵌入的有效运行。

\subsection{实验设置}

将输入分辨率从$60\times90$提升至$128\times192$
（空间面积从$5400$像素增至$24576$像素，$4.55\times$提升），
其余所有配置保持不变：
MambaVision-T骨干完全冻结，
仅训练输出投影层与时序Mamba模块；
训练数据、课程学习策略、学习率调度、评测协议均与上一阶段一致。
该设置确保分辨率是唯一的设计变量。

\subsection{速度段差异分析}

碰撞改善幅度呈现明显的速度段梯度：
低速段（$72.9\%$）$>$ 中速段（$49.5\%$）$>$ 高速段（$26.3\%$）。
该梯度源于以下两个因素的叠加。

第一，低速段对空间精度的依赖更强。
在$\SI{5}{m/s}$目标速度下，
无人机以较低速度穿越障碍场，
障碍物在视野中停留时间更长，
策略需要在多帧中持续判断障碍边界并规划精细的绕行轨迹。
$60\times90$分辨率下深层Stage的极端token退化
（Stage~4仅$2$个token）导致策略丧失精细的空间判别能力，
$128\times192$分辨率通过恢复空间采样密度直接缓解了这一瓶颈。

第二，高速段碰撞基数已低，边际改善空间受限。
$\SI{12}{m/s}$段在$60\times90$下的平均碰撞次数仅为$7.6$次，
碰撞密度本身已处于较低水平。
此时避障性能的瓶颈逐渐从空间分辨率转向时序决策精度与反应时间裕度，
分辨率提升能带来的边际收益相应减小。

关于$\SI{12}{m/s}$段SR从$100\%$降至$80\%$的现象，需谨慎解读。
高速段的动作裕度（action margin）极小，
微小的控制偏差即可导致任务失败。
更高的空间分辨率虽然提供了更丰富的输入信息，
但在骨干冻结的条件下，
网络对高频空间细节的处理能力有限，
额外的空间信息可能在个别试验中引入控制振荡。
此外，$80\%$与$100\%$的差异仅对应10次试验中的2次失败，
在小样本条件下存在固有的统计波动。
后续阶段的风险状态补分布蒸馏（\ref{sec:ch5_riskset}节）将专门解决高速段的稳定性问题。

\subsection{计算开销评估}

分辨率从$60\times90$提升至$128\times192$使输入像素总量增加$4.55\times$，
但推理延迟的增幅远小于像素增幅。
表~\ref{tab:ch5_resolution}的数据显示，
各速度段的推理延迟从约$\SI{20}{ms}$增至约$\SI{21}{ms}$，
平均增幅约$5\%$（$\SI{1}{ms}$量级）。
延迟增幅远低于像素增幅的原因在于：
MambaVision-T的主要计算量集中在深层Stage的token处理，
而非浅层的卷积嵌入；
输入分辨率翻倍主要影响Stem与Stage~1的计算量，
深层Stage的负载增长相对有限。

$\SI{21}{ms}$的推理延迟仍远低于$\SI{50}{ms}$的实时控制阈值（$\SI{20}{Hz}$控制频率），
满足部署要求。
综合碰撞降幅（$26\%$--$73\%$）与延迟增幅（$\sim5\%$），
分辨率提升是一项高收益--低开销的设计改进。

\subsection{小结}

本节的实验证据确认：
在MambaVision-T的分层卷积嵌入架构下，
$60\times90$的低输入分辨率构成空间表征能力的结构性瓶颈。
将分辨率提升至$128\times192$后，
各速度段碰撞事件次数下降$26.3\%$--$72.9\%$，
速度效率同步提升$4.6\%$--$32.1\%$，
$\SI{5}{m/s}$段SR从$80\%$恢复至$100\%$。
上述改善在骨干完全冻结的条件下取得，
排除了训练效果对性能变化的贡献；
推理延迟仅增加约$5\%$，
计算开销可忽略。

基于上述结论，
后续所有MambaVision实验锁定$128\times192$输入分辨率。

\section{视觉骨干受控解冻实验}
\label{sec:ch5_unfreeze}

\subsection{动机与控制变量声明}

前两节已依次锁定深度归一化口径（threshold模式，$\tau=0.09$）
与输入分辨率（$128\times192$），
消除了输入分布偏移与空间采样不足两个混淆因素。
在此基础上，
冻结骨干高分辨率MambaVision方案（以下简称"冻结高分辨率方案"）
虽已通过可行性验证，
但碰撞指标与ViT+Mamba基线仍存在差距。

一个自然的假设是：
MambaVision-T的ImageNet预训练权重在自然图像分布上训练获得，
与避障任务的输入分布（低纹理单通道深度图像）存在域间差异。
在骨干完全冻结的条件下，
预训练特征未经任务适配，
对深度图像中的特定视觉模式（近场障碍边界、无纹理深度梯度等）
的判别能力可能存在上限。
若能对骨干的高层Stage进行受控微调，
有望在保持预训练泛化基础的同时获得任务特异性适应。

本节实验在前两节锁定的归一化与分辨率配置基础上，
仅引入一个新的设计变量：
MambaVision-T骨干Stage~3是否参与训练。
其余所有配置（训练数据、课程学习策略、
投影层/时序模块/控制头的训练状态、评测协议）
均与冻结高分辨率方案保持严格一致。

"伪安全"现象形成明确对照：
该节中碰撞率的表面改善源于控制衰减导致的速度退化，
而本节中碰撞恶化发生在速度基本不变的条件下，

而非\ref{sec:ch5_norm}节所讨论的输入分布偏移伪象。

\subsection{原因讨论}

解冻Stage~3后碰撞指标全面恶化的现象
指向以下可能的机制，
但各归因的确定性程度不同，需区别对待。

训练数据规模不足（高置信度）。
当前训练集仅包含585条专家轨迹。
Stage~3解冻后新增的可训练参数
在有限数据下可能导致对训练分布特定模式的过拟合，
当闭环评测中策略进入训练未覆盖的状态时，
表现出退化的避障行为。
该归因与"碰撞恶化在所有速度段均出现"的观察一致——
过拟合的影响是全局性的而非速度段特异的。

弱对齐约束可能不足以防止特征偏移（中等置信度）。
本实验采用的L2正则（$\lambda=0.01$）为弱约束，
其设计意图是在允许一定适应的同时保持与冻结特征的对齐。
然而，任务驱动的梯度可能在训练过程中将Stage~3的特征表示
逐步拉离预训练空间，
导致下游时序模块接收到偏移的输入分布。
该假设可通过以下方式进行事后验证：
在训练过程中监测冻结骨干输出$f_{\text{frozen}}(x)$
与解冻骨干输出$f_{\text{unfrozen}}(x)$之间的余弦相似度，
若该指标随训练epoch持续下降，
则构成特征偏移的直接证据。
本实验未在训练日志中记录该指标，属于实验设计的不足，
后续工作中建议将此作为标准监测项。

单一解冻策略的探索局限（需明确声明）。
本实验仅尝试了一种解冻配置
（Stage~3全参数解冻 + 差异化LR + 弱L2约束），
实验覆盖的策略空间有限。
解冻在当前配置下失败并不等同于"微调不可行"，
而反映的是特定约束方式与数据条件下的失败模式。

\subsection{结论与局限性}

在当前实验条件下
（585条轨迹、行为克隆训练目标、弱L2对齐约束、Stage~3全参数解冻），
骨干解冻未带来稳定收益，
且在所有速度段均导致碰撞事件次数显著增加（$53\%$--$135\%$），
速度效率无对应提升。
因此后续实验默认采用骨干完全冻结的设定。

该结论的适用范围需明确界定。
以下方向在本实验中未被覆盖，
可能在不同条件下改变微调的可行性：
\begin{itemize}
  \item 参数高效微调：
    低秩适配（LoRA）、Adapter层或仅微调LayerNorm参数等方法
    可大幅减少可训练参数量，降低过拟合风险；
  \item 渐进式解冻：
    从最后一层开始逐层解冻（gradual unfreezing），
    配合逐步减小的学习率，
    可能比一次性解冻整个Stage更稳定；
  \item 更强的对齐约束：
    弹性权重巩固（EWC）或EMA教师约束
    可提供比L2正则更有效的特征空间锚定；
  \item 更大规模的训练数据：
    增加轨迹数量或引入数据增强
    可能为微调提供更充分的梯度信号覆盖。
\end{itemize}

上述方向作为未来工作的可能探索路线，
不影响本章后续实验的设定——
在已有数据与训练框架下，
骨干冻结是当前条件下最可靠的选择。

\section{结构改动与动作蒸馏实验}
\label{sec:ch5_structure_kd}

骨干解冻在当前条件下未能带来收益（\ref{sec:ch5_unfreeze}节），
本节继续在骨干完全冻结的前提下探索两条替代路线：
（1）修改特征池化结构以注入方向信息（特征条件池化）；
（2）通过教师动作蒸馏约束学生决策（Action KD）。
两条路线均沿用前文锁定的深度归一化配置（threshold，$\tau=0.09$）
与输入分辨率（$128\times192$），
仅改变网络结构或训练损失这一单一设计维度。

\subsection{特征条件池化}

\subsubsection{方法}

MambaVision骨干的输出特征经全局平均池化（GAP）后投影至时序模块。
GAP将空间信息完全压缩为一个全局向量，
不保留与飞行方向相关的空间结构。
基于此，本实验引入特征条件池化（Feature-Conditioned Pooling, FCP）：
在GAP之前，
利用本体状态（姿态四元数）对特征图施加空间注意力权重，
使池化结果保留方向敏感信息。

\subsubsection{定量结果与分析}

图~\ref{fig:ch5_panoramic_collision}(a)给出了FCP与冻结基线（冻结高分辨率方案）
在各速度段的碰撞事件次数对比。

在$\SI{5}{m/s}$低速段，
FCP的碰撞事件次数从约$19.6$降至约$16$（降幅约$18\%$），
表现出一定改善。
然而在中高速段，碰撞指标全面恶化：
$\SI{9}{m/s}$段从约$10$升至约$16.5$（增幅约$65\%$），
$\SI{12}{m/s}$段从约$5.6$升至约$12$（增幅约$114\%$）。
图~\ref{fig:ch5_panoramic_collision}(b)显示各速度段的Mean $V_x$在两种方法间几乎无差异
（$\SI{9}{m/s}$段均约$\SI{5.9}{m/s}$，
$\SI{12}{m/s}$段分别约$\SI{7.4}{m/s}$与$\SI{7.5}{m/s}$），
确认碰撞恶化并非速度变化的副产物，
而是避障控制能力本身的退化。

对该结果的一种可能解释是：
在当前数据规模与任务复杂度下，
引入显式方向偏置未能带来稳定收益。
时序Mamba模块可能已从历史观测序列中隐式提取了运动方向信息，
额外的空间偏置在部分速度段引入了冗余信号。
但需指出，本实验未对时序模块的隐状态进行分析
（如方向编码的可视化或消融），
上述解释仅为基于结果的推断而非经过验证的机制。
在实践层面，简单的全局聚合（GAP）在当前条件下已足够，
条件池化的引入未带来一致的性能提升。

\subsection{动作知识蒸馏}

\subsubsection{方法与理论解释}

动作知识蒸馏（Action KD）不修改网络结构，
而是在训练损失中增加学生与教师动作输出之间的约束：
\begin{equation}
  \mathcal{L}_{\text{total}} = \mathcal{L}_{\text{BC}} + \lambda_{\text{KD}} \| \hat{a}_{\text{student}} - a_{\text{teacher}} \|_2^2
\end{equation}
其中教师为基线ViT+Mamba策略（已训练完毕并固定权重），
$\lambda_{\text{KD}}$为蒸馏权重。
该方案不引入结构变化，
纯粹通过梯度信号约束学生行为向教师对齐。

从优化视角，上述复合损失的最优学生动作可解析求解。
对$\hat{a}_{\text{student}}$求导并令其为零：
\begin{equation}
  a^*_{\text{student}} = \frac{a_{\text{gt}} + \lambda_{\text{KD}} \, a_{\text{teacher}}}{1 + \lambda_{\text{KD}}}
  = a_{\text{gt}} + \frac{\lambda_{\text{KD}}}{1 + \lambda_{\text{KD}}} \left( a_{\text{teacher}} - a_{\text{gt}} \right)
\label{eq:kd_interpolation}
\end{equation}
其中$a_{\text{gt}}$为专家示范动作。
式~\eqref{eq:kd_interpolation}表明，
蒸馏损失的效果等价于将学生的优化目标
从纯专家动作$a_{\text{gt}}$
向教师动作$a_{\text{teacher}}$方向偏移，
偏移量$\Delta a = \frac{\lambda_{\text{KD}}}{1+\lambda_{\text{KD}}} (a_{\text{teacher}} - a_{\text{gt}})$
与教师--专家动作差异成正比。

该偏移在不同速度段具有不同的后果。
在低速段，避障的安全动作裕度（action margin）较宽，
微小的目标偏移不会使动作越过碰撞边界，
蒸馏约束可能起到正则化作用，平滑学生的动作输出。
在高速段，安全动作裕度极窄（微小的角速度偏差即可导致碰撞），
同样幅度的目标偏移更容易将动作推过决策边界，
导致碰撞率上升。

\subsubsection{高速段劣化的理论解释}

Action KD在高速段的碰撞增幅（$35\%$--$88\%$）
可由式~\eqref{eq:kd_interpolation}的目标偏移机制部分解释。
本实验中教师（ViT+Mamba，$60\times90$分辨率）
与学生（MambaVision，$128\times192$分辨率）
使用不同的视觉骨干与输入分辨率，
对同一深度场景产生的动作$a_{\text{teacher}}$
与专家标注$a_{\text{gt}}$之间存在系统性偏差。
蒸馏损失将该偏差按比例$\frac{\lambda_{\text{KD}}}{1+\lambda_{\text{KD}}}$
注入学生的优化目标。

在$\SI{12}{m/s}$高速段，
无人机以接近$\SI{7.5}{m/s}$的前向速度穿越障碍场，
单帧决策窗口约$\SI{20}{ms}$，
安全转向的角速度裕度极窄。
式~\eqref{eq:kd_interpolation}引入的目标偏移$\Delta a$
在此条件下更容易使学生动作越过碰撞边界，
导致碰撞率上升。
在$\SI{5}{m/s}$段，
较宽的动作裕度吸收了同等幅度的偏移，
蒸馏约束的正则化效应反而略有益处。

需指出，上述解释虽然与观测到的速度段梯度一致，
但本实验仅测试了单一$\lambda_{\text{KD}}$值，
未对不同蒸馏权重下的碰撞率变化进行扫描，
也未直接测量$\|a_{\text{teacher}} - a_{\text{gt}}\|$
在不同速度段的分布。
因此该归因属于"与理论预测一致的可能解释"，
而非经过完整验证的因果关系。

多数试验表现尚可，但少数试验出现灾难性碰撞事件。

而非模型容量或结构的整体缺陷——
若为容量问题，碰撞分布应呈现更均匀的整体恶化，
而非重尾的极端事件主导模式。

\subsection{保守归因与局限性}

特征条件池化与动作蒸馏两条路线均未在中高速段带来安全性改善，
且两者表现出共同的模式：
低速段可观测到边际收益，中高速段碰撞次数显著恶化，速度效率基本不变。
结合$\SI{12}{m/s}$段碰撞分布的重尾特性，
在当前实验空间内，
训练数据对风险状态（特别是高速段近碰撞窗口）的覆盖不足
成为最可疑的性能瓶颈因素。

该归因需明确其条件边界与不确定性。
以下因素在本节实验中未被覆盖，
可能影响上述两条路线的有效性：
\begin{itemize}
  \item FCP仅测试了一种条件信号（姿态四元数）与一种注意力机制，
    未探索更细粒度的空间偏置方案；
  \item Action KD仅测试了单一$\lambda_{\text{KD}}$值，
    未对蒸馏权重进行系统扫描；
  \item 未直接测量教师与专家动作在不同速度段的偏差分布，
    也未分析教师策略本身在高速段的稳定性；
  \item 未尝试更高级的蒸馏方案（如特征级蒸馏、渐进式蒸馏权重调度）。
\end{itemize}

因此，"数据覆盖不足"应被理解为当前实验证据所支持的最可疑假设，
而非排除其他可能的确定性结论。
基于该假设，下一阶段（\ref{sec:ch5_riskset}节）将直接验证：
定向补充风险状态样本能否突破当前性能瓶颈。

\section{风险状态补分布蒸馏}
\label{sec:ch5_riskset}

\subsection{核心假设}

前述实验未能达到预期性能，其主要原因可归结为数据分布的局限性，而非模型本身的容量不足。
具体而言，
行为克隆（BC）所依赖的训练数据主要由专家策略在正常飞行状态下生成，
极少包含近碰撞（near-collision）等边缘状态；
然而，在闭环评测过程中，
MambaVision学生策略由于累积的动作偏差，
不可避免地会偏离专家轨迹，进入训练数据未曾覆盖的未知状态区域。
在这些高风险状态下，
学生策略缺乏正确的行为指引，
导致碰撞概率急剧上升。

基于上述分析，提出如下核心假设：
无需修改模型结构或解冻骨干网络，
仅需定向补充高风险状态样本，并利用高质量的教师策略进行重新标注与微调，
即可有效突破当前的性能瓶颈。

\subsection{方法：风险状态采集与加权微调}

风险状态补分布蒸馏（Risk-set Distillation）的流程包含三个步骤：

步骤一：风险状态采集。
将当前MambaVision学生策略部署于Spheres环境中进行闭环飞行（类似DAgger的在线采集），
记录完整的观测--状态时间序列。
在采集数据中按以下规则标注风险窗口：
\begin{itemize}
  \item $d_{\min}$计算域：取归一化深度图像的$1\%$分位数作为$d_{\min,t}$。
    深度值在采集时已经过$d / d_{\text{divisor}}$归一化至$[0, 1]$区间
    （与网络输入口径一致），
    阈值判断亦在该归一化域上进行。
  \item 触发阈值：$d_{\min,t} < d_{\text{safe}} = 0.20$
    （归一化深度，固定常数）。
    该值对应物理距离约$\SI{2}{m}$--$\SI{3}{m}$
    （取决于环境深度量程与归一化除数$d_{\text{divisor}}$），
    为障碍物提供约$1$--$2$倍机身宽度的安全裕度。
  \item 风险窗口：以触发帧$t$为中心，取非对称窗口
    $[t - 20,\; t + 10]$（前20帧、后10帧），
    对应$\SI{30}{ms}$采样间隔下约$\SI{600}{ms}$前文与$\SI{300}{ms}$后文。
    前向窗口更长以捕捉趋近障碍的动作序列，
    后向窗口覆盖闪避后恢复阶段。
    相邻窗口间隔$\leq 5$帧时自动合并，避免高度重叠的冗余样本。
  \item 采集规模：由于本阶段的核心目标在于优化模型在高速场景下的表现，
    因此仅针对$\SI{9}{m/s}$与$\SI{12}{m/s}$两个高速段进行数据补充，
    各采集约30个回合的轨迹。
    此轮采集中低速段（$\SI{3}{m/s}$与$\SI{5}{m/s}$）未被纳入采集范围；
    如后续分析所示，这一偏向高速场景的实验设计，意外导致模型在低速场景下的性能出现退化。
\end{itemize}

步骤二：教师离线标注。
对采集到的风险状态，
由基线ViT+Mamba教师策略提供速度指令标注。
教师策略接收的深度图输入为学生采集时同步记录的原始深度帧，
经双线性下采样至教师的原生分辨率$60 \times 90$后推理。
尽管教师与学生的输入分辨率存在差异（$60 \times 90$ vs $128 \times 192$），
此处的定向校正仍然有效：
教师仅需在风险窗口内提供"朝哪个方向躲"的避障动作方向指引，
而非对全局空间细节的精确编码——
这与5.7节中全量动作蒸馏要求逐帧匹配教师精细控制量的场景本质不同\footnote{全量动作蒸馏需要教师对所有状态（包括常规飞行）提供高保真监督，
分辨率差异导致的系统性偏差在大规模样本中累积；
而风险窗口校正仅涉及少量近碰撞帧，教师的避障方向性信息对分辨率不敏感。}。

步骤三：加权微调。
在风险样本上对学生策略进行少量epoch的微调，
损失函数采用Huber损失（对离群值更鲁棒），
并对横向速度$v^y$与竖直速度$v^z$分量施加更高权重——
这两个分量直接决定横向闪避与纵向规避行为，
是近碰撞窗口中最关键的避障决策维度
（前向速度$v^x$主要由目标速度决定，对避障贡献较小）。
同时保留弱特征对齐约束，
防止微调偏离预训练特征空间。

\section{低速补强与最终模型综合评估}
\label{sec:ch5_lowspeed}

\subsection{“平滑性提升却导致碰撞率增加”的失败模式}

\subsubsection{基于最小深度的低速补强方案}

在首次针对低速场景（$\SI{5}{m/s}$）的补强采集中，本研究沿用了与高速段相同的策略：
即以学生策略在环境交互（Rollout）过程中最小深度观测值 $d_{\min}$ 低于设定阈值的状态作为风险触发条件，
收集对应时间窗口的数据交由教师策略重新标注并进行微调。
然而，在 $\SI{5}{m/s}$ 的低速飞行场景中，无人机在障碍物附近停留的时间相对更长，导致 $d_{\min}$ 阈值被频繁触发。
这使得大量“飞行状态安全但距离障碍物较近”的常规样本被错误地纳入了风险数据集，从而引入了严重的噪声干扰。

定量分析表明，V\_fail 版本在 $\SI{5}{m/s}$ 场景下的性能表现呈现出矛盾的特征：

但同时，其指令抖动 Jerk 却从 $0.08 \pm 0.02$ 大幅下降至 $0.01 \pm 0.00$（降幅 $87.5\%$），
平均飞行速度 $V_x$ 则从 $\SI{4.11}{m/s}$ 略降至 $\SI{3.99}{m/s}$（降幅 $2.9\%$）。

这种性能演变模式的特征十分鲜明：
尽管策略的控制输出变得极为平滑（Jerk 几乎趋近于零），但它实际上丧失了执行紧急转向以规避碰撞的能力。
当面临需要大机动避让的危急情况时，无人机倾向于选择更加平缓但存在较高碰撞风险的轨迹。

\subsubsection{失败案例的经验与启示}

上述失败案例通过定量的实验数据揭示了一条关键原则：
在基于视觉的自主导航任务中，控制平滑性不能单独作为衡量系统安全性的代理指标。
Jerk 降低 $87.5\%$ 的同时碰撞率却上升 $21.7\%$，这充分证明了“更平滑的轨迹”与“更安全的飞行”之间并不存在必然的因果关联。
这一反直觉的现象也为本章在性能评估体系设计中，仅将指令抖动（Jerk）作为评价控制品质的“补充参考指标”，而非“核心安全指标”提供了有力的实证支持。

\subsection{教师Omega事件触发的低速补强}

\subsubsection{关键改进}

基于上述失败分析，
最终的低速补强方案做了两项关键修正：

\begin{enumerate}
  \item 数据来源：从学生rollout切换为教师rollout。
    教师策略在低速段的飞行行为质量更高（几乎零碰撞），
    其风险窗口数据的标注也更可靠。

  \item 触发条件：从$d_{\min}$阈值切换为Omega事件（$\Omega$-event）触发。
    触发信号取自飞行记录中速度指令的第三分量$v^z_{\text{cmd},t}$
    （即策略输出$\mathbf{v}_t$的$v^z$维度）。
    当无人机执行紧急避障转向时，
    竖直速度指令会出现显著脉冲，
    因此$|v^z_{\text{cmd},t}|$的峰值可作为"转向决策关键帧"的可靠代理信号。

    $\Omega$-event的检测流程如下：
    \begin{enumerate}
      \item 对每个回合计算$|v^z_{\text{cmd},t}|$序列的$90\%$分位数作为自适应阈值$\tau_\Omega$
        （典型值约$0.73$，随回合难度自动调节）；
      \item 在满足$|v^z_{\text{cmd},t}| > \tau_\Omega$的候选帧中，
        筛选$[t-5, t+5]$窗口内的局部最大值；
      \item 相邻触发帧间设$10$帧去抖间隔（debounce，
        对应采样间隔$\SI{30}{ms}$下约$\SI{300}{ms}$），
        抑制同一次转向的重复触发。
    \end{enumerate}
    每个$\Omega$-event以触发帧为中心取$[t-20, t+10]$风险窗口
    （与$d_{\min}$模式相同的非对称窗口），
    窗口间隔$\leq 5$帧时合并。
    在$\SI{5}{m/s}$教师rollout（30个回合）中，
    每回合平均触发$7.6 \pm 1.4$个$\Omega$-event
    （最少4个，最多10个），
    共采集$7275$帧风险状态样本。
\end{enumerate}

\subsubsection{低速段修复效果}

Omega事件触发方案在$\SI{5}{m/s}$段取得了决定性改善：
碰撞率从V\_mid的$4.66\%$降至$0.56 \pm 0.31\%$（降幅$88.0\%$），
碰撞事件次数从$1.67$降至$0.27$（降幅$83.8\%$），
CPM从$0.09$降至$0.01$。
与此同时，Mean $V_x$从$\SI{4.11}{m/s}$维持在$\SI{4.09}{m/s}$（降幅仅$0.5\%$），
Jerk从$0.08$降至$0.05$——
控制平滑度有所改善，但远未退化到V\_fail的极端平滑（$0.01$），
保持了必要的紧急转向能力。

$\SI{3}{m/s}$段的改善更为彻底：
碰撞率从V\_mid的$2.68\%$降至$0.00\%$（零碰撞），
CPM从$0.05$降至$0.00$。
高速段指标在Omega补强后基本保持不变
（$\SI{9}{m/s}$碰撞率$2.13\%$ vs V\_mid $2.17\%$，
$\SI{12}{m/s}$碰撞率$1.34\%$与V\_mid持平），
确认低速补强未对已有的高速段成果造成扰动。

该版本即为第5章所有对比实验中的"MV+Mamba最终版"。

\section{第5章小结}

第5章通过逐步消融验证的方法学，
在保持控制闭环主体（时序Mamba模块、训练流程、部署协议）不变的条件下，
系统探索了MambaVision替代ViT视觉编码器的可行性与能力边界。
核心结论如下：

\begin{enumerate}
  \item 口径一致性是可信前提：
    深度归一化口径的不一致会导致"表面更安全实际更慢"的伪结论，
    所有消融对比必须在锁定口径下进行。

  \item 分辨率是MambaVision替代后的关键收益点：
    将输入分辨率从$60\times90$提升至$128\times192$后，
    MambaVision分层卷积嵌入的空间表征能力得到充分释放，
    碰撞率下降$26\%$--$73\%$。

  \item 盲目解冻与结构/蒸馏技巧引入"更激进或更平滑但更撞"的失败模式：
    骨干解冻、特征条件池化与动作蒸馏三条路线均未通过验证，
    但共同揭示了问题的根源在于数据分布而非模型容量或结构。

  \item 风险状态补分布蒸馏是突破点：
    高速段蒸馏使$\SI{9}{m/s}$碰撞率降低$27.7\%$、$\SI{12}{m/s}$降低$29.5\%$，
    为"数据分布覆盖不足是主要瓶颈"这一假设提供了强支持证据。
    同时揭示了速度段不对称效应——高速改善伴随低速退化，
    源于采集配置仅覆盖高速段。

  \item Omega事件触发是低速补强的正确方式：
    首次低速补强（$d_{\min}$触发）产生了"更平滑但更撞"的典型失败：
    Jerk降低$87.5\%$但碰撞率上升$21.7\%$，
    以定量证据证明平滑性不能作为安全性的代理指标。
    切换为教师rollout + $v^z$指令峰值事件（$\Omega$-event）触发后，
    $\SI{5}{m/s}$段碰撞率从$4.66\%$降至$0.56\%$（降幅$88\%$）。

  \item 最终模型安全性全面不低于基线：
    MV+Mamba最终版在Spheres环境所有5个速度段碰撞率均$\leq$基线，
    $\SI{12}{m/s}$高速段碰撞率仅为基线的$44.5\%$（$1.34\%$ vs $3.01\%$），
    同时飞行速度效率全面更高。
    在Trees（OOD）环境中，
    中高速段碰撞率降幅达$56\%$--$84\%$，
    Mean $V_x$提升$40\%$--$45\%$。
    推理延迟约$\SI{22}{ms}$，满足$\SI{20}{Hz}$实时闭环要求。
\end{enumerate}

局限性：
（1）MambaVision骨干完全冻结，未充分发掘任务自适应微调的潜力；
（2）风险状态的触发条件依赖启发式规则（$d_{\min}$阈值或$v^z$指令峰值），缺乏形式化的安全保证；
（3）所有实验在仿真环境中完成，Sim-to-Real迁移有待验证；
（4）教师--学生的分辨率差异限制了动作蒸馏的有效性，
更高分辨率的教师策略可能带来额外收益。
